# SideNote Markdown 전체 문법 테스트

이 문서는 SideNote WYSIWYG Preview가 지원하는 Markdown 요소를 한 번에 확인하기
위한 예시입니다. 노트의 Markdown source에 이 문서 내용을 붙여넣은 뒤 Preview로
전환해 각 섹션을 확인하세요.

---

## 1. 제목

# 제목 레벨 1
## 제목 레벨 2
### 제목 레벨 3
#### 제목 레벨 4
##### 제목 레벨 5
###### 제목 레벨 6

각 제목은 크기와 굵기가 서로 달라야 합니다.

---

## 2. 문단과 줄바꿈

빈 줄로 구분한 텍스트는 서로 다른 문단입니다.

이 문단의 첫 번째 줄입니다.
이 줄은 일반 개행으로 만든 soft line break이며 같은 문단 안에 표시됩니다.

이 줄 끝의 백슬래시는 강제 줄바꿈입니다.\
이 줄은 hard line break 다음에 표시됩니다.

---

## 3. 인라인 서식

일반 텍스트, **굵은 텍스트**, *기울임 텍스트*,
***굵고 기울인 텍스트***, ~~취소선 텍스트~~를 비교합니다.

문장 안의 `const value = 42;`는 인라인 코드입니다.

서식을 중첩한 **굵은 글자 안의 *기울임***과
~~취소선 안의 **굵은 글자**~~도 표시되어야 합니다.

이스케이프한 \*별표\*, \# 샵, \[대괄호\]는 서식이 아닌 문자로 표시됩니다.

## 10. 작업 목록

- [ ] 완료하지 않은 작업
- [x] 완료한 작업
- [X] 대문자 X로 표시한 완료 작업
- [ ] 상위 작업
    - [x] 완료한 하위 작업
    - [ ] 완료하지 않은 하위 작업

Preview에서 체크박스를 클릭하면 화면 상태와 Markdown의 `[ ]` 또는 `[x]`가 함께
바뀌어야 합니다.

## 12. 표

| 왼쪽 정렬 | 가운데 정렬 | 오른쪽 정렬 | 기본 정렬 |
| :--- | :---: | ---: | --- |
| 왼쪽 | 가운데 | 오른쪽 | 기본 |
| **굵게** | *기울임* | `inline code` | ~~취소선~~ |
| `A \| B` | [링크](https://example.com) | 숫자 123 | 한글과 English |

헤더와 셀 테두리, 각 열의 정렬, 셀 내부의 인라인 서식을 확인합니다. 셀 사이의
1px 구분선은 라이트 모드에서 검은색, 다크 모드에서 흰색으로 보여야 합니다.

## 13. 코드 블록

### 언어가 있는 여러 줄 코드 블록

```javascript
function greet(name) {
  const message = `Hello, ${name}!`;
  console.log(message);
}

greet('SideNote');
```

언어 이름과 복사 버튼, JavaScript 문법 강조, 줄 번호 정렬을 확인합니다.

### text 코드 블록

```text
첫 번째 코드 줄
두 번째 코드 줄
세 번째 코드 줄
```

### 언어가 없는 한 줄 코드 블록

```
single line code block
```

한 줄 코드 블록에는 줄 번호가 표시되지 않아야 합니다. 언어가 없는 블록의
헤더에는 `text`가 표시되지만 저장된 여는 fence에는 언어가 추가되지 않아야
합니다.

### 코드 fence를 포함한 코드 블록

````markdown
```javascript
console.log('nested fence');
```
````

## 14. WYSIWYG 상호작용 체크리스트

- [ ] 빈 문단에서 `# `부터 `###### `까지 입력해 각 제목으로 변환되는지 확인
- [ ] 빈 제목에서 `Backspace`를 눌러 일반 문단으로 돌아가는지 확인
- [ ] 빈 문단에서 `> `, `- `, `1. `, `` ``` ``, `---` 입력 규칙 확인
- [ ] 목록에서 `Enter`로 다음 항목 생성, `Shift+Enter`로 항목 내부 줄바꿈 확인
- [ ] 두 번째 이후 목록 항목에서 `Tab`, 중첩 항목에서 `Shift+Tab` 확인
- [ ] 빈 목록 항목에서 `Enter`를 눌러 목록을 종료하는지 확인
- [ ] URL을 붙여넣으면 `[URL](URL)` 형태로 저장되는지 확인
- [ ] 코드 블록 헤더의 언어를 클릭해 수정하고 `Enter`로 저장되는지 확인
- [ ] 코드 블록의 복사 버튼과 여러 줄 코드의 줄 번호 정렬 확인
- [ ] 문서를 더블클릭하면 같은 위치의 전체 Markdown source가 열리는지 확인
- [ ] source에서 `Ctrl+Enter` 또는 `Cmd+Enter`로 Preview에 돌아오는지 확인
- [ ] custom 제목 수정 중 `Escape`를 누르면 제목 수정만 끝나고 노트가 열린 상태로 유지되는지 확인
- [ ] 실행 취소와 다시 실행 단축키 확인

---

## 15. 예상되는 제한

SideNote는 보안을 위해 원시 HTML 렌더링을 사용하지 않습니다. HTML 태그가 필요한
기능은 Markdown 요소나 에디터 확장 API로 구현해야 합니다.
